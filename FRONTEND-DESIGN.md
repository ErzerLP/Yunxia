## 1. 设计原则与风格

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **效率优先** | 减少点击次数，常用操作一步可达 |
| **上下文感知** | 右键菜单、悬浮工具栏，操作不离手 |
| **反馈即时** | 按钮 hover、加载骨架屏、操作 toast，不让用户猜测 |
| **一致性** | 所有页面共享相同的设计语言、间距、圆角、动效 |
| **容错性** | 危险操作二次确认、撤销提示、优雅降级 |

### 1.2 视觉风格

| 维度 | 选择 |
|------|------|
| **设计语言** | 现代扁平 + 微质感（参考 Linear / Vercel / Cloudreve V4） |
| **圆角** | 全局 8px（按钮/卡片/输入框），标签 4px，头像 50% |
| **阴影** | 极轻微（`shadow-sm` / `shadow-md`），拒绝厚重阴影 |
| **边框** | 1px `border-border`，深色模式 `border-border/50` |
| **间距** | 8px 基准倍数：xs=4, sm=8, md=16, lg=24, xl=32 |
| **字体** | 系统字体栈：`"Inter", "PingFang SC", "Microsoft YaHei", sans-serif` |
| **字号** | 正文 14px，标题 18-24px，辅助文字 12px |
| **行高** | 正文 1.5，标题 1.3 |

### 1.3 色彩系统

**浅色模式**：

| Token | 值 | 用途 |
|-------|-----|------|
| `--background` | `#ffffff` | 页面背景 |
| `--foreground` | `#171717` | 主文字 |
| `--muted` | `#f5f5f5` | 次要背景、hover |
| `--muted-foreground` | `#737373` | 次要文字 |
| `--border` | `#e5e5e5` | 边框 |
| `--primary` | `#2563eb` | 主品牌色（蓝色） |
| `--primary-foreground` | `#ffffff` | 主色上的文字 |
| `--accent` | `#f5f5f5` | 强调背景 |
| `--destructive` | `#ef4444` | 危险操作（删除） |
| `--success` | `#22c55e` | 成功状态 |
| `--warning` | `#f59e0b` | 警告状态 |

**深色模式**：

| Token | 值 | 用途 |
|-------|-----|------|
| `--background` | `#0a0a0a` | 页面背景 |
| `--foreground` | `#fafafa` | 主文字 |
| `--muted` | `#171717` | 次要背景 |
| `--muted-foreground` | `#a3a3a3` | 次要文字 |
| `--border` | `#262626` | 边框 |
| `--primary` | `#3b82f6` | 主品牌色（稍亮） |

**已确认：品牌主色蓝色系（#2563eb）**
- 专业、可信，Cloudreve/OneDrive/Linear 验证
- 拒绝渐变色，保持简洁

## 2. 设计决策（已确认）

以下 7 个关键交互决策已确认，前端设计基于此方案冻结：

### 决策 A：全局布局模式

| 选项 | 描述 | 适用场景 |
|------|------|---------|
| **A1. 左侧固定边栏**（推荐） | 左侧窄边栏（64px 图标 / 200px 展开），右侧主内容区。类似 Google Drive / Dropbox。 | 桌面端，导航项较多 |
| **A2. 顶部导航栏** | 顶部水平导航，下方全宽内容。类似传统管理后台。 | 导航项少，强调内容 |
| **A3. 可折叠边栏** | 默认窄边栏，hover 或点击展开。节省空间。 | 小屏幕桌面端 |

**已确认：A1（左侧固定边栏，64px 图标模式，可展开 200px）**
- 文件管理类产品的标准布局，用户零学习成本
- 边栏容纳存储源列表、主要导航
- 给文件浏览区域最大空间

---

### 决策 B：文件浏览区域布局 ✅

| 选项 | 描述 |
|------|------|
| **B1. 面包屑 + 文件列表** ✅ | 顶部面包屑导航，下方文件列表。无目录树。类似 Google Drive。 |
| B2. 左侧目录树 + 右侧文件列表 | 双栏布局。 |
| B3. 三栏布局 | 信息密度高但复杂。 |

**已确认：B1（面包屑 + 文件列表）**
- 云存储产品的现代标准
- 目录树在 Web 端占用过多水平空间
- 如需目录树，P1 可增加"显示目录树"开关

---

### 决策 C：文件打开方式 ✅

| 选项 | 描述 |
|------|------|
| **C1. 单击打开，Ctrl+单击多选** ✅ | 单击打开文件夹/预览文件；Ctrl/Cmd 多选；Shift 范围选择。 |
| C2. 双击打开，单击选中 | Windows 传统交互。 |
| C3. 长按/右键菜单打开 | 移动端风格。 |

**已确认：C1**
- Google Drive、Dropbox、Notion 等现代 Web 应用均采用此模式

---

### 决策 D：文件预览方式 ✅

| 选项 | 描述 |
|------|------|
| **D1. 右侧抽屉预览** ✅ | 右侧滑出抽屉显示预览和详情。不离开当前页面。 |
| D2. 中央弹窗预览 | 中央弹窗+遮罩。 |
| D3. 新页面/全屏预览 | 全屏或新页面打开。 |

**已确认：D1（右侧抽屉，360px）**
- 不离开当前上下文，可同时浏览其他文件
- 抽屉内可显示文件详情和视频播放

---

### 决策 E：上传面板位置 ✅

| 选项 | 描述 |
|------|------|
| **E1. 底部固定浮动面板** ✅ | 底部浮动面板，可收起为图标。 |
| E2. 右侧侧边栏 | 上传任务在右侧边栏。 |
| E3. 顶部横幅 | 顶部进度条。 |

**已确认：E1**
- 不占用主要内容区域，可随时查看进度

---

### 决策 F：动效风格 ✅

| 选项 | 描述 |
|------|------|
| **F1. 精致微动效** ✅ | 150-250ms，有目的性的动效。类似 Linear/Vercel。 |
| F2. 极简无动效 | 瞬间切换。 |
| F3. 丰富动效 | 弹性动画、stagger 入场。 |

**已确认：F1**
- 提升感知性能，反馈操作结果，但不过度

## 4. 页面详细设计

### 4.1 登录页 (/login)

**功能**：用户登录、初始化向导跳转（首次启动）

**布局**：
- 居中卡片，宽度 400px
- 上方产品 Logo + "云匣"名称 + 副标题 "你的私有云存储"
- 下方："首次使用？系统将引导你完成初始化"

**元素**：
- 用户名输入框（带图标）
- 密码输入框（带显示/隐藏切换）
- 登录按钮（主色，全宽）
- 错误提示（红色，输入框下方）

**动效**：
- 页面加载：卡片从下方滑入 + fade in（300ms，ease-out）
- 登录成功：按钮显示 spinner → 页面 fade out → 跳转
- 登录失败：输入框左右 shake（300ms），错误提示 fade in

**交互**：
- Enter 键提交
- 密码输入框支持显示/隐藏
- 登录错误 5 次后显示"请 15 分钟后重试"，按钮 disabled

---

### 4.2 初始化向导 (/setup)

**功能**：首次启动时创建管理员账号

**步骤**：
1. **欢迎页**：产品简介 → "开始"
2. **创建管理员**：用户名、密码、确认密码
3. **完成**："进入云匣"按钮

**交互**：
- 步骤指示器（顶部进度点）
- 下一步按钮验证通过后才可用
- 完成后自动登录并跳转到文件页

## 7. 动效设计

### 7.1 动效原则

| 原则 | 说明 |
|------|------|
| **有意义** | 每个动效都服务于理解（不是装饰） |
| **快速** | 150-250ms，不让用户等待 |
| **自然** | 使用 ease-out / ease-in-out，拒绝线性 |
| **克制** | 同一时间最多一个动效 |

### 7.2 具体动效

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| **页面加载** | 内容 fade in + 轻微上移（10px） | 200ms | ease-out |
| **抽屉滑出** | 从右侧 translateX(100%) → 0 | 250ms | cubic-bezier(0.32, 0.72, 0, 1) |
| **抽屉关闭** | translateX(0) → 100% + 遮罩 fade out | 200ms | ease-in |
| **文件列表加载** | 骨架屏 → 内容 fade in | 150ms | ease-out |
| **文件项 hover** | 背景色渐变过渡 | 100ms | ease |
| **文件选中** | 边框颜色 + 背景色渐变 | 100ms | ease |
| **文件删除** | 行高收缩 + fade out（slide up） | 200ms | ease-in |
| **文件上传完成** | 新行 slide down + fade in | 200ms | ease-out |
| **Toast 出现** | 从上方滑入 + fade in | 200ms | ease-out |
| **Toast 消失** | fade out + 上移 | 150ms | ease-in |
| **弹窗出现** | 遮罩 fade in + 内容 scale(0.95→1) + fade in | 200ms | ease-out |
| **弹窗消失** | 内容 scale(1→0.95) + fade out + 遮罩 fade out | 150ms | ease-in |
| **批量操作栏** | 从底部 translateY(100%) → 0 | 200ms | ease-out |
| **上传面板展开** | 高度展开 + fade in | 200ms | ease-out |
| **侧边栏展开** | 宽度 64px → 200px，内容 fade in | 200ms | ease-out |
| **主题切换** | 全局颜色过渡 | 200ms | ease |
| **目录切换** | 文件列表 cross-fade（旧 fade out，新 fade in） | 150ms | ease |
| **搜索聚焦** | 搜索框宽度展开 | 200ms | ease-out |
| **右键菜单** | scale(0.95→1) + fade in | 100ms | ease-out |
| **按钮点击** | scale(1→0.97→1) | 100ms | ease |
| **加载 Spinner** | 旋转 360° 无限循环 | 1s | linear |
| **进度条** | 宽度平滑过渡 | 300ms | ease-out |
| **空状态出现** | 插画 + 文字 stagger fade in | 300ms | ease-out |

### 7.3 动效实现建议

- 使用 CSS `transition` 和 `animation`（避免 JS 动画库增加包体积）
- 复杂动画可使用 Framer Motion（已包含在 React 生态中）
- 使用 `will-change: transform, opacity` 优化性能
- 支持 `prefers-reduced-motion` 媒体查询，为不适用户户关闭动效
