# Trash API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 Yunxia 后端补齐 Draft 回收站闭环：删除到回收站时持久化元数据，并提供 list / restore / delete one / clear source API。

**Architecture:** 保留现有真实存储语义：文件或目录先移动到 `/.trash/...`，再新增 `trash_items` 元数据表作为回收站真相源。`FileService` 负责在 trash 删除成功后写入记录；新增 `TrashService` 负责列表、恢复、单项永久删除和清空 source 回收站。local 与 S3 统一走应用层编排，底层仍复用 local 文件系统与已注册 storage driver 的 move/delete 能力。

**Tech Stack:** Go 1.25、Gin、GORM、SQLite、现有 local/S3 storage driver、httptest
