# ACL API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 Yunxia 后端补齐 Draft ACL 规则管理接口，支持管理员对 ACL 规则执行 list / create / update / delete。

**Architecture:** 这一轮只实现 ACL 规则管理面，不把 ACL 判定接入文件访问主链路。新增 `acl_rules` 表作为规则真相源；新增 `ACLService` 做校验与 CRUD 编排；新增 `ACLHandler` 提供 admin-only HTTP 接口。为避免前端阻塞，先支持 `subject_type=user`、`effect=allow/deny`、路径精确过滤。

**Tech Stack:** Go 1.25、Gin、GORM、SQLite、httptest
